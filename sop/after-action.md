# After Action — 收官流程

> 每次重要任務結束後跑一次。做了不記 = 沒做。

---

## 什麼時候觸發

- 完成一個重要決策
- 結束一個 project 的階段
- 做完一次重要會議或活動
- 踩了一個坑、學到一個教訓
- 任何「以後會想回頭看的事」

---

## 三步收官

### Step 1：寫 Memory

在 `memory/` 建一筆紀錄：

```
# [簡短標題]

日期：YYYY/MM/DD

## 做了什麼
[一兩句話]

## 決策 / 結論
[做了什麼決定、為什麼]

## 教訓（如果有）
[踩了什麼坑、下次怎麼避免]
```

### Step 2：更新相關文件

這次的結果有沒有影響到 Vault 裡的其他文件？

- 決策改變了方向 → 更新 `context/`
- 學到新模式 → 更新 `identity/`
- 認識重要的人 → 更新 `people/`
- 項目進度推進 → 更新 `projects/`

### Step 3：更新 Memory Summary

如果夠重要，寫進 `memory-summary.md` 對應的表格。

---

## Agent 的角色

Agent 主動提醒跑收官流程，特別是：
- 剛做完重要決定但沒寫下來
- 一個 project 告一段落但沒更新 Vault
- 提到了教訓但沒有記錄

提醒方式：「這個要不要記一下？我幫你寫進 memory。」
